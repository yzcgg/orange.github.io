Redis

入门概述

Redis: Remote Dictionary Server(远程字典服务器)
基于内存的, 用C语言写的, 数据库.

Redis之父, 安特雷兹.

如果Redis挂了, 大量的访问又打到MySQL了, 所以要设置集群, 一个挂了另一个顶上.

讲了Redis7新特性:

  • 说Redis function不学, 还是用lua脚本
  • listpack代替zlist
  • 多个aof
  • 性能提升

安装redis, 刚开始装在Linux课给的公共服务器上发现不行, 没权限.
于是装在虚拟机上了.

更改redis.conf
老师有个好习惯, 他把官网的配置文件备份了一份, 不直接在原文件上修改

redis.conf配置文件，改完后确保生效，记得重启，记得重启

1 默认daemonize no 改为 daemonize yes 后台启动:是

2 默认protected-mode yes 改为 protected-mode no 保护模式关闭, 可以远程访问.

3 默认bind 127.0.0.1 改为 直接注释掉(默认bind 127.0.0.1只能本机访问)或改成本机IP地址，否则影响远程IP连接

4 添加redis密码 改为 requirepass 你自己设置的密码

我的密码是: zizhugongyuan 紫竹公园

redis: 退出是quit
关闭服务是: SHUTDOWN 也可以远程关闭 redis-cli -a 你的密码 shutdown

string 讲了一些基础命令

set 参数 ex:设置过期时间 xx: 存在则创建 nx: 不存在则创建
get: 先get到老值, 再set新值 keepttl: 把原有的过期时间保留着.
mset, mget
setnx, setex 这两个是分布式锁那里要用.
其实笔记上和官网都有.

讲了list, 就是队列的应用场景, 微信公众号我关注了100个博主, 我的页面总有一个最近更新队列, 就可以用list实现.

讲了hashmap, 应用场景是购物车(简单的实现), 确实能对应上, 相当于是表中有表, 每个用户对应一个hashmap, 每个hashmap的key是商品, value是商品数量, 因为redis的key-value可以是任意数据类型, 所以还可以保存用户id等等

list有序, 有重复. set无序无重复.

set应用场景:
微信抽奖小程序
展现点赞列表统计
可能认识的人

zset 的 score 可以是小数, 但是会丢失精度

bitmap

hyperloglog 用于统计 uv = unique visitor 只能返回有多少个不同的元素, 而不能保存每个元素具体是什么. 有0.81%误差 是string类型

GEO 是 zset类型 , 用于附近的人, 滴滴, 等移动互联网场景.

redis - stream
就是redis版本的MQ 消息中间件
大概有个印象, 最后还是用kafuka(也不见得)

bitfield 能修改每一位, 案例演示了吧hello –> hxllo的过程, 通过二进制更改0和1实现.

2024-4-6 今天看到了27, 十大数据类型看完了, 除了5个基础的, bitmap, GEO, hyperloglog, 这三个还有点儿用, stream和bitfield先不做重点.
