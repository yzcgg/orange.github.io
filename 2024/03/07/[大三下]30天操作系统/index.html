计算机系统结构01--基础知识

16位寄存器:

  • AX accumulator 累加寄存器 x是extend
  • CX counter
  • DX data
  • BX base 基址寄存器
  • SP stack pointer
  • BP base pointer
  • SI source index 源变址寄存器
  • DI destination index 目的变址寄存器

8位寄存器

  • AL L:low
  • CL
  • DL
  • BL
  • AH H:high
  • CH
  • DH
  • BH

32位寄存器就是在16位寄存器前面加一个E

  • EAX … 剩下的不一个一个写了

段寄存器, 16位, segment register

  • ES extra segment 附加寄存器
  • CS code segment 代码寄存器
  • SS stack 栈寄存器
  • DS data 数据段寄存器
  • FS 没有名字
  • GS 没有名字

也就是说第一个扇区的512个Byte是BIOS通电后自动就映像到内存中的, 也叫启动区, 利用启动区的512B的空间写一个IPL(initial programme loader), 把剩下的磁盘扇区也映射到内存中.

今天先读了磁盘, 就是把磁盘的内容映像到内存.

扇区是从1号开始的, 所以要从2号开始.

先初始化了一堆寄存器, 其中代表 柱面 磁头 扇区的最好记住
然后读第一个扇区, 中间有一小段试错的代码, 然后读到18号扇区, 换一个磁头, 再读, 又读完了, 磁头切换回0, 进入下一个柱面, 重复. 也就是说, 一共读了(如果算上启动区的话): 18 * 512 * 2 * 10 = 184320 B

上图中, 第二个扇区映像到内存中的位置从0x08200而不是0x08000开始, 据B站一个up说是因为想让1号扇区看起来像是加载到0x08000位置(尽管其实不是)

512用16进制表示是0x0200, 也就是比如从0x07c00开始读一个扇区, 要读到0x07dff

内存管理模块

data – 模拟数据

属性:

  • 数据内容. for example: 长度为40的string.
  • 数据大小. for example: 大小是10 B.

方法:

  • access(start, offset)

    计算step, step的含义即一个字节能表示多少字.
    access的start和offset都说的是字节.
    比如我们要访问10 ~ 30 字节, 我要先算出来, 这些字节表示了多少个字, 然后再把string中的这些字显示出来.

page – 模拟内存页

这里, 有点儿难看懂

其实是这样的:

  • 首先content是一个字典, key是首地址, value 是 上面定义的data对象.
  • 那么, content.get就得到了一个data对象, data.access获得相应的数据(string)
  • 最后为什么access的参数是: offset - self.usedAddress[sflag][0], length 呢?
    • 那是因为要换成相对的, 对于一个string来说, 开始就是0, 长度就是string.len. 而data的access方法的参数是start(字节), offset(字节). 对于一个页面来说, 我offset - start得到我的相对偏移量, 然后访问字节的长度是length.
  • 总之最终返回的是string.

memory – 模拟内存

内存中有128页, 每页16kb. 通过一个列表memory表示, 页表中的每个元素都是page对象.

access 和 write 是只能对 单页 进行.
malloc 是可以分配多个页面.
